Alasan pemilihan teknologi ini karena Laravel dan Vue.js memiliki ekosistem yang mature, dokumentasi lengkap, dan performa yang baik untuk aplikasi enterprise."

**Q3: "Bagaimana sistem menangani keamanan data?"**
> Jawaban: "Sistem menerapkan beberapa layer keamanan, antara lain: autentikasi berbasis token (Laravel Sanctum), role-based access control (RBAC) untuk mengatur hak akses, validasi input di sisi frontend dan backend untuk mencegah SQL injection dan XSS, serta password di-hash menggunakan bcrypt. Untuk deployment production, akan ditambahkan SSL/HTTPS."

**Q4: "Apakah sistem sudah diujicoba oleh user?"**
> Jawaban: "Sistem telah dilakukan user acceptance testing (UAT) dengan owner dan staff Vassion Bali menggunakan data periode November-Desember 2025. Feedback yang diterima positif, terutama terkait kemudahan penggunaan dan kecepatan generate laporan. Beberapa minor bug yang ditemukan sudah diperbaiki."

**Q5: "Bagaimana jika listrik mati saat input data?"**
> Jawaban: "Sistem menggunakan database transaction, sehingga jika terjadi kegagalan di tengah proses, data akan di-rollback dan tidak akan menyebabkan data corrupt. Untuk kedepannya, bisa ditambahkan fitur auto-save draft dan backup otomatis ke cloud storage."

**Q6: "Apa perbedaan sistem Anda dengan aplikasi sejenis seperti Accurate atau Jurnal?"**
> Jawaban: "Sistem yang saya kembangkan lebih customized untuk kebutuhan spesifik Vassion Bali, dengan fokus pada integrasi penjualan dan inventory management. Biaya operasional juga lebih rendah karena tidak ada subscription fee. Namun, untuk fitur-fitur advanced seperti payroll atau multi-currency, memang belum tersedia dan bisa menjadi pengembangan selanjutnya."

**Q7: "Bagaimana scalability sistem jika bisnis berkembang?"**
> Jawaban: "Sistem dirancang dengan arsitektur modular menggunakan Laravel dan Vue.js, sehingga mudah untuk di-scale. Database menggunakan relational model yang normalized, sehingga performa tetap optimal meskipun data bertambah. Jika traffic meningkat, bisa di-deploy ke cloud server dengan auto-scaling seperti AWS atau Google Cloud."

**Q8: "Kenapa tidak menggunakan mobile app, hanya web?"**
> Jawaban: "Saat ini sistem berbasis web responsive yang bisa diakses via mobile browser. Alasannya: development time lebih cepat (cukup satu codebase), maintenance lebih mudah, dan tidak perlu download/install app. Namun, untuk kedepannya bisa dikembangkan Progressive Web App (PWA) atau native mobile app jika dibutuhkan."
